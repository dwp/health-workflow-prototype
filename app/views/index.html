Task based workflow for HCPs - August 2025 - v1

In design

Problem statement

*Needs updating

In 2023 Chaffinch were assigned work to look into how task management could be implemented in the Health Assessment Service (HAS), with the intention that this would be a new way of working, and to eventually replace the current status-based approach. The first stage of this has been completed and the Review further medical evidence task is now live.

This new approach aims to improve how information and evidence is received, processed through HAS and subsequently how it is allocated to a user, in order to process the referral efficiently, reducing cost, time and also to reduce the need for existing admin workarounds.

Link to detailed problem statement (opens in new tab)

Hi-level design decisions

  • Integration of allocting a task and 'open tasks'
  • Table updates to include 'Due in' to give users an idea of priority
  • Notification if a new task has come in for a case previously worked on
  • Option to mark multiple tasks as complete
  • Confirmation for marking tasks as complete

Appointee details - April 2025 - vxx

In development

Problem statement

If someone claiming PIP or WCA is incapable of managing their own affairs due to mental incapacity or severe physical disability, then an appointee may be assigned. The appointee will act on behalf of the claimant for benefit purposes.

An appointee can be an individual, or an organisation.

However, there have been instances where the claimant has been contacted rather than the appointee.

Agents need to access various systems using manual processes in order to find out if an appointee has been assigned to a referral.

This is time consuming for the agent. It opens up for error and risks contact with the incorrect person.

This costs the department money and causes lack of trust

Hi-level design decisions

  • Addition of 'Appointee' section into 'Claimant details'
  • Addition of 'Appointee' into side nav
  • Use of 'Warning text' in 'Personal details' when appointee is registered
  • Appointee details are displayed to agent in 'Appointee' section
  • 'No appointee' message added to 'Appointee' section when no appointee is registered
