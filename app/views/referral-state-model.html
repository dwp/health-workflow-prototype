Referral state Model

This prototype shows the minimum viable product (MVP) using a referral state model with the aim of simplifying how a referral can be pushed end-to-end through the Health Assessment Service, with a benefit-agnostic approach. It allows a user to do the following:

Prototype

  • view a list of referrals that have come into the health assessment service via the referral gateway
  • Show benefit type so Case Manager can identify a WCA referral as these have a different clerical process in place
  • Show referral date so a Case Manager can identify whether a referral needs chasing
  • perform necessary identity checks
  • move the referral to the next stage by having the ability to update the status of a referral as it moves through the assessment process
  • start assessment if the referral is in 'paper based review' or 'assessment booked'
  • view report if the referral is in 'awaiting decision' or 'done'
